Unified Login-Style Workspace

Single, consistent interface for Login, Dashboard, Settings, Blog, Category, Admin, and Super Admin panels. All protected content and uploaded assets are surfaced through this unified portal.

Login

JWT-based authentication; employees get view-only access; admins and super admins inherit elevated permissions.

Dashboard

Landing shell for authenticated users; links into module cards using the same login-style layout.

Settings

Admins and super admins can read/update allowed settings via /api/settings.

Blog

Editable content for admins and super admins via /api/blog; employees have read-only visibility if exposed.

Category

Hierarchical catalog. Super admins can CRUD; admins and employees consume read-only data.

Admin Panel

Admin scope to manage employees and view categories, audiences, settings, and blog updates.

Super Admin Panel

Full control over users, categories, audiences, blog, and settings with audit-friendly JSON stores.

Role Permissions

  • Employee: login only; view permitted content.
  • Admin: manage employees; edit blog; update settings; categories read-only; no audience management.
  • Super Admin: full CRUD on users, categories, and audiences; edit blog and settings; end-to-end control.
