Admin & Super Admin Panels

Login-style shell that unifies admin and super admin workflows: employee management, audiences, categories, settings, and blog.

Dashboard Settings Blog Category Home

Admin Scope

  • Create, update, and deactivate Employees via /api/users/emp.
  • View all users; cannot create Admins.
  • Edit blog and settings; categories are read-only.
  • No audience management permissions.

Super Admin Scope

  • Full CRUD on users including Admin creation via /api/users/admin.
  • Full category CRUD and audience management.
  • Edit blog and settings with elevated privileges.
  • Gateway for future modules with RBAC expansion.

Audiences

Attach multi-level audiences to categories. Super Admin controls assignments; Admins have visibility only.

  • POST /api/audiences/ to add to a category.
  • GET /api/audiences/categories/:path/audiences to inspect.
  • DELETE /api/audiences/{id}/category/:path to detach.

Settings & Blog

Login-style content editors surfaced through the unified UI.

  • Settings: /api/settings (Admin, Super Admin).
  • Blog: /api/blog (Admin, Super Admin).
  • Employees inherit view-only; no mutations.
